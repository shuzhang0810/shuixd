    BRIEF

    For me, user experience design is about creating experience people love. It’s no need to ask people to learn complex interaction or force them to walk through intensive tutorials. An interface should be self-explanatory and intuitive. Depending on varied context and business goals, interaction and interface can be dramatically different. Nothing is an all-round solution. Besides, I always believe that creating a little surprise is also an important part of a product as it simply makes people happy.

    Signup Redeisgn

    This project aims to smooth the experience when users try to log in and create a new account. Bounce rate on register dropped from 25.1% to 2.2% and 40% to 1% for mobile users in the first month after launch. Registration success rate from 14% to 19%. The average time for registration reduced 27% (29 seconds less than before).

    Specifically, there are a couple issues identified in redesign work. Login and registration page lack feedbacks. In others words, users do not get enough information to move forward. Also, there is a lot of room to help users to correct unexpected case. Besides, registration form asks too many info from users which are not necessary. Furthermore, app users who try to sign up will be redirected to a web page which is not mobile friendly. This creates a gap in experience for mobile users. Last but not least, the user interface is outdated. After identified issues above, a research was conducted. User behavior was carefully analyzed based on data from usage tracking tool. Sketches, mockups, and interactive prototypes were created and tested.

    Widescreen

    Project Widescreen was initiated to make the entire App Annie site and platform responsive/scale to a user’s screen size in order to best utilize their window size and offer a better overall user experience while using our products.

    The App Annie platform was originally designed and developed as a center-aligned, fixed-width dashboard platform. Although supported data source and new feature sets have evolved dramatically over the past few years, the web UI has not followed suit as quickly. As our users’ screen sizes have gotten larger over time, the side margins that used to serve as a nice frame for our products now feel like underutilized screen real estate.

    Search Optimization

    The goal of this project is to optimize the search function and help users to easily navigate to particular content. This is a concept of possible enhancement.

    Original search does not support live search, which means no suggestion shows up while users typing in the search box. Besides, search results page shows items by app platform. Users can only see search results from App Store, Google Play, Amazon etc. Last but not least, information architecture is clear enough to help users to index content.

    Watch App

    The goal of this project is to use the latest technology to glue the gaps between mobile phone users and web users. I’m a dedicated UX designer in a 3 people team. This project wins the champion of 2015 App Annie Hackathon.

    This project is not only considered about smoothing the user experience from desktop to mobile and watch but also conveying the branding to audiences. Functional Apple watch demo was designed and built a month after Apple Watch launch. New technology is introduced and properly applied into the watch app, like hand-off, notification, and quick summary.
